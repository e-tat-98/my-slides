# スライド構成テンプレート

発表形式に応じたスライドの枚数・順序・パターンの選び方。

## LT（ライトニングトーク）5分

目安: **6〜8枚**

```
1. タイトルスライド（〜30秒）
   → タイトル + バッジ（技術スタック等）+ 登壇者名

2. 自己紹介（〜30秒、省略可）
   → 名前 + アイコン + 一言

3. 課題・背景（1〜2枚、〜1分）
   → 「こういう問題がありました」
   → Before パターン or 問題点リスト

4. 解決策・内容（1〜2枚、〜2分）
   → 「こうやって解決しました」
   → コードブロック / After パターン / アーキテクチャ図

5. まとめ（1枚、〜30秒）
   → 3点以内のキーポイント（左ボーダーカード）

6. END
```

## 技術セッション 15〜30分

目安: **15〜25枚**

```
1.  タイトルスライド
2.  自己紹介
3.  Agenda（Toc コンポーネント）
--- セクション区切り ---
4.  セクション1 ブレイク
5.  内容スライド × 3〜5枚
--- セクション区切り ---
6.  セクション2 ブレイク
7.  内容スライド × 3〜5枚
--- セクション区切り ---
8.  セクション3 ブレイク（デモ / コード / 実績等）
9.  内容スライド × 2〜3枚
---
10. まとめ
11. 参考リンク・資料（省略可）
12. END
```

## チュートリアル / ハンズオン

目安: **20〜30枚**、コードスライド多め

```
1.  タイトル
2.  ゴール・前提条件（箇条書き or アイコングリッド）
3.  Agenda
---
4.  環境セットアップ（コードブロック中心）
5.  Step 1（説明 → コード → 実行結果の3点セット）
6.  Step 2
7.  Step 3 完成版
    → Before/After パターンで変化を示す
---
8.  まとめ・次のステップ
9.  参考リンク
10. END
```

## ストーリー構成の原則

1. **課題→解決→証明** の流れを作る（LT でも必ず課題から入る）
2. 1スライド1メッセージ：伝えたいことを1つに絞る
3. `v-click` で段階的に情報を開示し、聴衆の理解を待つ
4. コードスライドは Before/After か diff 形式で変化を見せる
5. まとめは最重要ポイント3点以内
6. セクションが3つ以上あるなら必ず Agenda スライドを入れる
